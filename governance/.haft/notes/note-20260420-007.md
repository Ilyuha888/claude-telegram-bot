# _justCleared flag in kill() prevents auto-resume overriding /new

## Rationale

tryAutoResume() only checked this.sessionId === null, which is also true after /new. Added private _justCleared = false; set to true in kill(), consumed (read+reset) at the top of tryAutoResume(). In-memory flag — doesn't survive process restart, which is correct: /new + bot-crash-before-next-message is not a real use case and the 24h TTL covers it anyway. Alternative (persisting cleared_at to session file) was rejected because prob-20260420-007 explicitly constrained "no schema change to /tmp/claude-telegram-session.json".

## Evidence

Deployed 2026-04-20 16:39. journalctl confirms clean startup. Observation indicator from prob-20260420-007 fired in production (user sent /new, next message auto-resumed a 3h-old PDF session). Fix addresses the false-positive auto-resume path.
