# No voice transcription signal in agent context — voice messages treated as typed text

## Signal

Telegram bot transcribes voice messages via OpenAI Whisper and passes the resulting text to Claude as a plain user message. No metadata flag indicates the message originated from speech. Agent treats the transcription as carefully-composed text, causing two failure modes: (1) interprets filler words, incomplete sentences, and verbal shortcuts literally; (2) asks for clarification on intent that was clear from speech context. buildMessageContext (dec-20260420-002) already enriches message context for forward/reply/quote provenance — voice transcription is the same pattern but missing. Detailed spec: Agent_Obsidian_Vault/tasks/bot-improvement/2026-04-17-telegram-transcription-handling.md

## Problem Type

synthesis

## Constraints

- Voice message path in linuz90/claude-telegram-bot is known: Whisper API call in media handler, result passed to session.sendMessageStreaming
- No new external dependencies — enrichment is text prepend only, same pattern as dec-20260420-002
- Must not affect typed-text messages — conditional on voice origin flag only
- Transcription confidence score should be included if available from Whisper response

## Optimization Targets

- Agent interprets voice intent rather than literal transcription text
- Clarification requests on voice input decrease significantly

## Observation Indicators

- Clarification request rate on voice messages (don't optimize — just watch)
- False-positive intent interpretation (agent guesses wrong intent)

## Acceptance

buildMessageContext prepends a transcription notice when message source is voice (detected by bot, not by Claude). Agent interprets informal/garbled voice input for intent rather than literal wording ≥90% of the time without asking for clarification. Verified by: sending 5 voice messages with deliberate filler words and one STT error; confirming agent responds to intent, not literal text.

## Blast Radius

linuz90/claude-telegram-bot voice handler + buildMessageContext helper (dec-20260420-002). Vault and Claude Code runtime untouched.
