# Bot message formatting and Telegram UX quality of life

## Signal

The Telegram bot currently sends plain text replies with no formatting conventions. Real-world usage reveals UX gaps: no distinction between informational responses and action confirmations, no structured formatting for vault search results, permission dialogs render as raw text, no message length management for long responses, and no conventions for code blocks, links, or structured data in Telegram's limited markdown subset. The bot also has no greeting or help message for new sessions.

## Constraints

- Telegram Bot API markdown subset (MarkdownV2 or HTML)
- Must work on mobile (small screen)
- No custom bot UI — all formatting via message text only

## Optimization Targets

- Readable, well-formatted bot responses in Telegram
- Clear visual distinction between query results, confirmations, and status messages

## Observation Indicators

- Message length distribution
- User re-asks due to unclear formatting

## Acceptance

Bot uses consistent formatting for: search results (bulleted with note paths), confirmations (clear yes/no prompt), status messages (emoji prefix), long responses (chunked or summarized). User can scan responses without reading walls of text.

## Blast Radius

Vault CLAUDE.md (formatting instructions), possibly ~/.claude/CLAUDE.md on VM

## Reversibility

high
