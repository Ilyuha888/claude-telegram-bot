# No LLM provider abstraction decision recorded

## Signal

ARCHITECTURE.md references Z.ai and an LLMClient interface, but no ADR or quint decision exists for the LLM provider layer. Token budget management, retry strategy, fallback providers, streaming support, and cost controls are all undecided. The name "Z.ai" appears without explanation or rationale.

## Constraints

- Runtime LLM must not get arbitrary shell access (ADR-005)
- Provider interface must be placed behind an adapter (project convention)
- Must support single-user MVP token budget without complex billing infrastructure

## Optimization Targets

- Define a minimal, swappable LLMClient interface before writing any tool-call logic
- Decide retry/fallback policy explicitly rather than ad-hoc

## Observation Indicators

- Streaming support requirements
- Cost per 1M tokens for candidate providers
- Rate limit characteristics

## Blast Radius

High — the LLM client is the core intelligence boundary of the platform. Every tool call, vault write, and response generation passes through it. Provider lock-in, API contract, and error handling patterns set here will be hard to change later.

## Reversibility

low
