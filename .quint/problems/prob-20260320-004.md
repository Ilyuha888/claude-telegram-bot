# ADR-008 locks session schema fields too early with internal contradictions

## Signal

ADR-008 specifies exact session state fields before any implementation exists. The field `active_facts` is typed as a free-form string list, violating the ADR's own rule about typed boundaries. The field `decisions` name-collides with the quint artifact namespace. Premature field locking without a running system invites schema churn.

## Constraints

- Session schema must use explicit typing and Pydantic (project convention)
- Field names must not collide with established quint or domain vocabulary
- Must not store secrets in session state

## Optimization Targets

- Defer field specifics until a working prototype informs what fields are actually needed
- Eliminate the active_facts free-form string anti-pattern

## Observation Indicators

- How many ADR-008 fields survive to a first working prototype unchanged

## Blast Radius

Medium — session state schema is internal to the runtime. Changing it before any code is written is low-cost, but if accepted as-is it will constrain future Pydantic model design and create confusion with the quint `decisions/` directory.

## Reversibility

high
