# Dulwich worktree support unverified against ADR-006 requirements

## Signal

~~ADR-006 required worktrees for pre-approval staging. That requirement is superseded by dec-20260320-005 (V2), which eliminates pre-approval worktrees entirely.~~

The remaining open question: dec-20260320-005 (V2) applies `change_manifest` blobs in-memory using Dulwich's ObjectStore and index API without checking out a working tree. This pattern must be verified against Dulwich's actual API before implementation. If in-memory commit is not viable, the fallback is a minimal short-lived worktree checked out only at commit time (seconds, not review duration).

## Constraints

- No subprocess or shell execution in the runtime path (ADR-005)
- Git operations must remain auditable and policy-controlled
- Must actually work on the target platform (macOS + Linux)

## Optimization Targets

- Confirm or deny Dulwich worktree capability with a minimal proof-of-concept
- Identify the least-invasive resolution if Dulwich is insufficient

## Observation Indicators

- Dulwich issue tracker / changelog for worktree support
- GitPython as fallback alternative

## Blast Radius

High — if Dulwich cannot satisfy ADR-006 worktree requirements, either ADR-005 (no shell), ADR-006 (staging model), or ADR-009 (Dulwich) must be revised. All three are accepted invariants.

## Reversibility

medium
