# ADR-006 staging complexity unjustified for single-user MVP

## Signal

ADR-006 specifies a 6-state + 4-transition Git approval state machine for vault changes. This was designed defensively but the platform has exactly one user with no concurrent editors and no merge conflicts possible. The complexity cost (implementation, testing, debugging) is paid upfront against a risk that does not exist at MVP scale.

## Constraints

- Must preserve the review-before-commit safeguard (no silent vault mutations)
- Must not require shell/subprocess for Git operations (ADR-005)
- Single-user constraint must remain explicit

## Optimization Targets

- Minimize implementation surface for MVP
- Keep the approval UX simple enough for one user to review vault diffs over Telegram

## Observation Indicators

- State machine reachability — how many states are actually reachable in single-user operation
- Lines of code required to implement vs. simpler alternative

## Blast Radius

High — ADR-006 is the foundation of the vault write path. Changing it cascades into ADR-009 (Dulwich), the staging service, the Telegram approval UX, and the Git commit flow.

## Reversibility

medium
