# Sync vs async runtime model undecided

## Signal

ADR-002 specifies Python 3.12+ but is silent on sync vs async execution model. Every service boundary (Telegram handler, vault writer, scheduler, LLM client) depends on this choice. The current architecture diagrams in ARCHITECTURE.md mix patterns without committing.

## Constraints

- Must not introduce shell execution or subprocesses into the runtime LLM path (ADR-005)
- Must be compatible with chosen Telegram library (ADR-003)
- Must support single-user MVP scope without over-engineering for concurrency

## Optimization Targets

- Minimal cognitive overhead for a single-user workload
- Clear, consistent pattern across all service layers

## Observation Indicators

- Event loop nesting complexity
- Third-party library async compatibility surface

## Blast Radius

High — affects every service boundary, handler design, concurrency primitives, and third-party library selection across the entire runtime.

## Reversibility

low
