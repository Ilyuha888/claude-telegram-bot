# Sync vs async runtime model undecided

## Signal

ADR-002 specifies Python 3.12+ but is silent on sync vs async execution model. Every service boundary (Telegram handler, vault writer, scheduler, LLM client) depends on this choice. The current architecture diagrams in ARCHITECTURE.md mix patterns without committing.

## Constraints

- Must not introduce shell execution or subprocesses into the runtime LLM path (ADR-005)
- Must be compatible with chosen Telegram library (ADR-003)
- Must support single-user MVP scope without over-engineering for concurrency

## Optimization Targets

- Minimal cognitive overhead for a single-user workload
- Clear, consistent pattern across all service layers

## Observation Indicators

- Event loop nesting complexity
- Third-party library async compatibility surface

## Blast Radius

High — affects every service boundary, handler design, concurrency primitives, and third-party library selection across the entire runtime.

## Reversibility

low

## Characterization v1 (2026-03-20)

| Dimension | Role | Scale | Unit | Polarity | Measurement |
|-----------|------|-------|------|----------|-------------|
| Pattern consistency | target | ordinal | uniform / mixed / fragmented | higher_better | Count of distinct concurrency patterns coexisting in the codebase (1=uniform, 2=mixed, 3+=fragmented) |
| Library compatibility friction | target | ordinal | low / medium / high | lower_better | Number of bridging shims required to satisfy the PTB v22+ async constraint across the call stack |
| Cognitive overhead (single-user MVP) | target | ordinal | low / medium / high | lower_better | Number of distinct mental models a contributor must hold to add a feature (sync vs async boundary, executor usage, thread safety) |
| Blocking-call exposure in hot path | target | ordinal | none / isolated / pervasive | lower_better | Whether blocking calls can stall the event loop during a typical message-handling turn |
| Reversibility cost | target | ordinal | low / medium / high | lower_better | Estimated effort to swap the concurrency model after first service layer is implemented |

**Parity rules:** All variants evaluated under single-user MVP load, Python 3.12+, python-telegram-bot v22+ (asyncio-native, non-negotiable), Dulwich (sync) for Git operations. No variant may change these constraints.
