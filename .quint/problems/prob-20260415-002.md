# Vault CLAUDE.md needs revision after real-world use

## Signal

The vault CLAUDE.md was written before the assistant went live. First real-world tests revealed gaps: Claude didn't auto-commit after writes (now fixed with confirm flow), permission prompts leaked into Telegram (fixed with defaultMode override), and the navigation guide was written from structure analysis without testing actual assistant queries. The CLAUDE.md likely needs tuning for: search strategies that actually work via Telegram, better instructions for handling Cyrillic path search, clearer agent zone conventions, and response formatting guidance.

## Constraints

- Do not break existing commit-confirm flow
- Keep CLAUDE.md under 200 lines

## Optimization Targets

- Claude follows vault CLAUDE.md instructions reliably on first try
- Vault queries via Telegram return useful results without manual guidance

## Observation Indicators

- Frequency of user corrections to Claude vault behavior

## Acceptance

Run 10 representative vault queries via Telegram (search, cross-reference, create note, edit note). Claude handles all correctly without user intervention beyond the confirm step.

## Blast Radius
