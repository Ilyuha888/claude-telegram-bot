# V3 — Layered: typed stub + Pydantic schema contract tier

## 1. Problem Frame

**Signal:** No ADR or architecture document specifies how to test code paths that invoke the LLM. The options — live API calls, recorded cassettes, mock adapters, deterministic stubs — have meaningfully different trade-offs in cost, flakiness, and coverage fidelity. Without a declared strategy, test files will be written inconsistently: some will hit the live API, some will skip the LLM layer entirely. This must be decided before any tool-call or handler code is written, not retrofitted later.

**Constraints:**
- Must not require live API calls in CI (cost and flakiness)
- Must not require shell execution or external services in unit tests
- LLMClient adapter boundary must be testable in isolation

## 2. Decision

**Selected:** V3 — Layered: typed stub + Pydantic schema contract tier

V3 is the only non-dominated variant. It matches V1's fast CI and asyncio compatibility while eliminating V1's high divergence risk via shared Pydantic schemas validated in the contract tier. It avoids V2's asyncio patching fragility and cassette maintenance cost. The contract tier runs as a required pre-merge gate, not in the default unit suite, so cost is controlled. The Protocol + Pydantic foundation is a stepping stone to V2-style cassette replay if ever needed.

**Invariants:**
- LLMClient is always accessed through LLMClientProtocol — no code may instantiate the concrete client directly in tests
- No test may call the live API unless it carries @pytest.mark.live
- Pydantic schemas are the single source of truth for tool-call request/response payloads — both production code and stubs validate against the same models

**Pre-conditions:**
- [ ] LLMClientProtocol defined before first handler or orchestrator code is written
- [ ] StubLLMClient placed in tests/stubs/llm.py
- [ ] Contract tests placed in tests/contract/
- [ ] CI pipeline has two stages: pytest -m 'not live' (every push) and pytest -m live (required pre-merge gate with OPENAI_API_KEY and OPENAI_BASE_URL secrets)

**Post-conditions:**
- [ ] prob-20260320-007 is Addressed
- [ ] ARCHITECTURE.md contains a Testing Strategy subsection
- [ ] All LLM-dependent orchestrator tests inject StubLLMClient via LLMClientProtocol
- [ ] Contract test suite validates real API responses against Pydantic schemas

**Admissibility:**
- NOT: No unittest.mock.patch directly on OpenAI SDK internals
- NOT: No cassette files committed to the main test suite
- NOT: No live API calls in the default CI unit run (pytest -m 'not live')

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| V3 — Layered: typed stub + Pydantic schema contract tier | **Selected** | V3 is the only non-dominated variant. It matches V1's fas... |
| V1 — Protocol fake only | Rejected | High divergence risk: fake silently drifts from real API wire format with no automated cross-check. Acceptable only as a fallback if the contract tier proves too expensive. |
| V2 — VCR cassette recording | Rejected | Eliminated by the asyncio compatibility constraint: httpx/aiohttp + vcrpy requires patching event loop internals, which is fragile. Cassette maintenance burden is also high. |

**Weakest link:** Stub fixture scripts need manual updates when tool schemas evolve; schema validation only catches drift when the contract tier is actually run (pre-merge gate, not every push)

**Evidence requirements:**
- StubLLMClient correctly reproduces ChatCompletion and tool-call response shapes
- Contract test suite runs green against live Gemini endpoint
- pytest -m 'not live' completes without any HTTP calls (verified via socket-level assertion or httpretty)

## 4. Consequences

**Rollback plan:**
Triggers:
- Contract tier proves too expensive to maintain due to schema churn
- Pydantic schema validation adds unacceptable implementation overhead
Steps:
1. Remove tests/contract/ directory
2. Remove @pytest.mark.live stage from CI pipeline
3. Keep LLMClientProtocol and StubLLMClient unchanged — interface does not change
4. Update this decision record to note demotion to V1
Blast radius: Low — production code is unchanged; only the contract test tier is removed

**Refresh triggers:**
- OpenAI SDK async interface changes (e.g. httpx transport layer rewrite)
- Gemini drops OpenAI-compatible endpoint
- Tool-call schema evolves in a way that requires >2 Pydantic model updates per quarter

**Affected files:** src/llm/protocol.py, src/llm/client.py, tests/stubs/llm.py, tests/contract/test_llm_contract.py, ARCHITECTURE.md
