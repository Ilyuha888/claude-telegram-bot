# ADR-007 scheduler magic numbers lack justification

## Rationale

ADR-007 specifies `min_interval=6h`, `max_runs=30`, and `expiry=30d` with no derivation. These read as arbitrary defaults, not as values calibrated to actual usage patterns or resource constraints. No user behavior data, no load analysis, and no cost model backs them. The numbers will be treated as authoritative by any implementer who reads the ADR without this context. They should be revisited once a prototype exists and real usage data is available.

## Evidence

ADR-007 text states the values as requirements without a "rationale" section explaining the derivation.
