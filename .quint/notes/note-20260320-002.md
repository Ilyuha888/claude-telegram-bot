# ADR-004 workspace lifecycle (topic deletion, archival) is unaddressed

## Rationale

ADR-004 defines a Telegram topic as a workspace namespace, which is correct. However, it does not specify what happens when the Telegram topic is deleted, archived, or when the user leaves the group. The vault state associated with that workspace, any in-progress Git staging, and any scheduled tasks all have undefined behavior at topic end-of-life. This is a gap that will surface in production even for a single user.

## Evidence

ADR-004 defines workspace creation semantics but has no section on workspace termination or archival.

## Affected Files

- `docs/adr/ADR-004-telegram-topic-workspace.md`

## Deprecated (2026-04-15)

**Reason:** ADR-004 workspace lifecycle was muscat-specific. Workspace concept now handled by Claude Code sessions + Telegram.
