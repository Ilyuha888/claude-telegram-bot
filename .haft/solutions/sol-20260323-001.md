# Solutions for: ADR-008 locks session schema fields too early with internal contradictions

Problem: prob-20260320-004

## Variants (3)

### V1. A — Fix-and-keep

Patch ADR-008 in place. Type active_facts as list[ContextFact] with id/category/value/source_turn_id. Rename decisions → workspace_rulings. Keep all fields. Maintain schema-first approach.

**Strengths:**
- Full type safety from day one
- Naming collision eliminated
- Design intent communicated clearly

**Weakest link:** Speculative field shapes lock design before any runtime feedback

**Risks:**
- Fields may not survive prototype — schema churn likely
- active_facts structure is speculative without usage data
- Higher rework if prototype needs different fields

**Rollback:** Amend ADR-008 again or supersede with a new decision

### V2. B — Defer-to-prototype

Supersede ADR-008 with principles only: schema must be versioned, typed (no bare strings), no namespace collisions. Remove specific field list from ARCHITECTURE.md. Define concrete schema during implementation.

**Strengths:**
- Maximum flexibility during prototyping
- No speculative field commitments
- Lowest rollback cost

**Weakest link:** No concrete schema reference for implementers to build against

**Risks:**
- Loss of design communication — implementer starts from zero
- Stable fields like schema_version and workspace_profile lose their documented shape
- May lead to ad-hoc schema decisions during implementation

**Rollback:** Add fields back when prototype stabilizes

### V3. C — Hybrid: skeleton + deferred

Keep stable fields (schema_version, workspace_profile, rolling_summary, pending_tasks, open_loops, open_artifacts, last_compacted_at) — these have clear semantics regardless of prototype outcome. Rename decisions → workspace_rulings (provisional). Defer active_facts entirely — it has no clear typed shape yet. Record principles for future field additions: must be typed, no namespace collisions, versioned.

**Strengths:**
- Retains design communication for stable fields
- Eliminates naming collision
- Removes speculative active_facts without losing proven fields
- Low rollback cost — only provisional fields need revision

**Weakest link:** workspace_rulings name is provisional — may need revision when runtime decisions semantics are clearer

**Risks:**
- workspace_rulings name is provisional and may still change
- Deferred fields could be forgotten

**Stepping stone:** yes — opens future possibilities

**Rollback:** Amend decision if prototype reveals different field needs; blast radius limited to session manager module
