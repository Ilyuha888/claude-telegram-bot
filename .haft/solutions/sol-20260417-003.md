# Solutions for: Vault CLAUDE.md needs V-A methodology contract

Problem: prob-20260417-002

## Variants (3)

### V-A. Full inline V-A section

Add a ~33-line 'V-A Methodology' section directly to CLAUDE.md covering all 5 elements: frontmatter schema (4 fields + value enums), note types, lifecycle states (draft→review→evergreen→archived), capture pipeline (3 steps), topic query order (MOC→wikilinks→mgrep), and MOC creation rule. No external files. CLAUDE.md ends at ~190 lines.

**Strengths:**
- Zero file dependency — rules always in scope
- Meets all 3 acceptance criteria directly
- No proactive-read rule needed

**Novelty:** Complete self-contained contract — agent never needs to read another file to know V-A rules

**Weakest link:** Line budget — 10 lines remaining; any future friction signal requiring >10 lines forces a section cut or externalisation

**Risks:**
- Tight headroom (10 lines) limits future iterative fixes
- If methodology evolves, changes spread across a dense section

**Diversity role:** High correctness, high budget cost

### V-B. Thin rules in CLAUDE.md + va-contract.md

Add ~10 rules-only lines to CLAUDE.md (frontmatter fields, 4 behavioral rules, pointer to spec). Create Agent_Obsidian_Vault/rules/va-contract.md with full definitions (unconstrained). Add 1 proactive-read entry in existing Proactive Zone Reads section. CLAUDE.md ends at ~172 lines.

**Strengths:**
- 28 lines headroom for future friction signals
- va-contract.md evolves freely without touching CLAUDE.md budget
- Established pattern from dec-20260416-002

**Novelty:** Externalises definitions, keeping CLAUDE.md as pure policy layer — mirrors how dec-20260416-002 externalised navigation into vault-index.md

**Weakest link:** File dependency — if agent doesn't load va-contract.md, behavioral rules work but type/lifecycle definitions are absent, edge cases misfire

**Risks:**
- Two-file dependency; silent degradation if proactive read fails
- Increases session context size slightly

**Stepping stone:** yes — opens future possibilities

**Stepping-stone basis:** Opens path to a richer va-contract.md that grows into templates, examples, and decision trees without ever touching CLAUDE.md

**Diversity role:** High correctness via two-file, strong headroom

### V-C. Compressed inline only (~12 lines)

Maximum compression: frontmatter schema as a single pipe-delimited line, lifecycle and capture pipeline as 4 dense rules, MOC rule as 1 line. All 5 elements present but lossy. CLAUDE.md ends at ~169 lines.

**Strengths:**
- 31 lines headroom
- Zero file dependency
- Fast to write and review

**Novelty:** Bets that the agent infers missing detail from context rather than explicit rules — minimal CLAUDE.md footprint

**Weakest link:** Correctness — compression drops nuance (user review step implicit, MOC creation threshold unstated); agent makes inference errors on ambiguous edge cases

**Risks:**
- Acceptance criterion 'without explicit instruction' may fail on edge cases
- Compressed rules are brittle — easy to misread

**Diversity role:** Minimal footprint, medium correctness

## Pareto Front Trade-Offs

- Full inline V-A section — Full correctness, zero dependency — pays with tight headroom (10 lines). Right if budget pressure is not expected.
- Thin rules in CLAUDE.md + va-contract.md — Full correctness + strong headroom (28 lines) — pays with one file dependency. Right if iterative friction-signal fixes are expected.
- Compressed inline only (~12 lines) — Best headroom (31 lines), zero dependency — pays with medium correctness. Right only if acceptance criteria can be relaxed or agent inference is trusted.

**Selection policy:** Maximize agent correctness first (acceptance criterion: agent applies V-A rules without explicit instruction); maximize line headroom second (room for future friction signal fixes); minimize file dependency third.

**Recommendation (advisory):** Thin rules in CLAUDE.md + va-contract.md

**Recommendation rationale:** V-B edges V-A under the declared policy. Correctness is tied (both HIGH) \u2014 V-B achieves it via a proactive-read rule, which is already a proven pattern (Proactive Zone Reads section handles tasks/, profile/ correctly). The 28 vs 10 line headroom difference is material: the vault CLAUDE.md iterated twice from Telegram friction signals and will keep iterating. 10 lines will close fast. The file dependency risk is mitigated because the proactive-read mechanism is working. Va-contract.md becomes a stepping stone for templates and richer agent instructions without ever touching the CLAUDE.md budget again.
