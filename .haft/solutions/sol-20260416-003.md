# Solutions for: Vault CLAUDE.md needs revision after real-world use

Problem: prob-20260416-002

## Variants (3)

### V-A. Inline Routing Table + Agent Zone Guide

Replace vague 'MOC-first + Glob/Grep' navigation with a bilingual routing table mapping common English query terms to Cyrillic vault paths. Fix tool references from Glob/Grep to mgrep. Add a 5-line Telegram context block (primary interface, message length awareness). Add a comprehensive agent zone workflow section covering notes/, drafts/, indexes/, tasks/, reviews/, profile/ — with use-case triggers, naming conventions, and lifecycle rules. Drop stale note count.

**Strengths:**
- All changes in one file — no two-file dependency
- Directly addresses gaps 1-6 including full agent zone workflow
- Deterministic routing — no reliance on mgrep semantic quality

**Novelty:** Locus of navigational knowledge: inline lookup table embedded in CLAUDE.md itself

**Weakest link:** Static routing table — vault restructuring silently invalidates path mappings

**Risks:**
- Routing table is a static snapshot — drifts silently as vault grows
- Pushes close to the 200-line budget (~185 lines)
- Agent zone guide is spec without enforcement

**Rollback:** Revert CLAUDE.md to previous version via git revert

### V-B. Thin CLAUDE.md + Generated Index + Agent Zone Bootstrap

Keep CLAUDE.md thin (~130 lines): rules only — commit-confirm flow, tool names (mgrep), Telegram context, safety rules, agent zone workflow guide. Move all navigational knowledge into Agent_Obsidian_Vault/indexes/vault-index.md — a machine-generated file listing every folder, its script type (Latin/Cyrillic), English aliases, and note count. CLAUDE.md instructs: read vault-index.md before any search. The index is bootstrapped on first run and can be regenerated when the vault structure changes.

**Strengths:**
- CLAUDE.md stays focused on rules and workflows — not navigational snapshots
- Index is regenerable — drift is fixable rather than silent
- Bootstraps agent zone with a real first artifact, establishing the pattern for notes/tasks/reviews
- Solves 200-line budget pressure by externalizing navigational content

**Novelty:** Locus of navigational knowledge: separate living document in agent zone, regenerable independently of CLAUDE.md

**Weakest link:** vault-index.md availability — missing or stale index degrades all search to unguided mgrep

**Risks:**
- Two-file dependency: if vault-index.md is missing or stale, navigation degrades to blind mgrep
- Adds one-time index generation step before CLAUDE.md is fully operational
- Index maintenance requires explicit regeneration — not automatic

**Stepping stone:** yes — opens future possibilities

**Stepping-stone basis:** The vault-index.md establishes Agent_Obsidian_Vault/indexes/ as the first real agent artifact. This pattern becomes the foundation for richer agent zone content: topic summaries, link graphs, per-domain indexes.

### V-C. Semantic Search-First + Agent Zone Guide

Rewrite the navigation section around mgrep as the primary tool, trusting its semantic capability to bridge Cyrillic/English ambiguity without an explicit routing table. Provide a short fallback chain: (1) mgrep with natural-language query, (2) broaden with Russian synonyms if no results, (3) list role MOCs as last resort. Fix tool refs, add Telegram context block, add full agent zone workflow section. No routing table, no separate index file.

**Strengths:**
- Most compact variant (~160 lines)
- No routing table to maintain
- Aligns with mgrep as the system's canonical search tool

**Novelty:** Locus of navigational knowledge: delegated to mgrep's semantic model at query time — no static mapping maintained

**Weakest link:** mgrep semantic quality for Cyrillic path bridging — unknown; no evidence it handles 'databases' → 'БД/' reliably

**Risks:**
- Entirely dependent on mgrep's Cyrillic/Latin semantic matching quality — untested for this vault
- No deterministic fallback if mgrep returns poor results for Cyrillic paths
- Agent zone bootstrapping is deferred
