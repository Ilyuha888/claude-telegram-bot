# Missed dimension on prob-20260416-006: scheduled fire must not interrupt an active dialogue

## Rationale

All three Pareto-front variants fire a proactive Telegram message into the same chat where the user holds conversations. If a dialogue is in progress (session streaming or recent user message), the scheduled message lands mid-thread and breaks the conversational context. This is a UX regression not captured in the compare. V_cron_cli (external curl) cannot detect dialogue state without IPC and is effectively blocked from solving this. V_inprocess can gate the fire on session state — it lives inside the process that owns the ClaudeSession. This dimension further differentiates the variants and should be added to the characterization before a decision is recorded.
