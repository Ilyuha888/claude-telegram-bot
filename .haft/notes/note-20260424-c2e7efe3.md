# dec-20260424-aa4e9e2d amendment — add [New session] action to notification keyboard

## Rationale

Notification inline keyboard expands from [Show][Delete][Remind later] to [Show][New session][Delete][Remind later]. [New session] behavior: (1) call session.clear() — same as /new, fresh context; (2) call sendMessageStreaming() with the notification's full content prepended as a priming user turn ("Here is my [routine name]: [full_content]"); (3) Claude responds, digest is now in session history; (4) user continues conversationally ("postpone task 3", "promote that draft"). This bridges Path A isolation (delivery doesn't pollute working session) with conversational actioning (user opts in when ready). Notification remains in notifications.json until explicitly [Delete]d — [New session] does not auto-delete.
