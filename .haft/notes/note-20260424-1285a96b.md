# Notification system design decided — soft delivery, notifications.json store, Mode-2 Notifications tab

## Rationale

User confirmed (2026-04-24): scheduled fires use Path A (ephemeral Claude session, out-of-band delivery — no context pollution of active working session). Delivery is Option C soft-first: compact Telegram message + inline keyboard [Show] [Delete] [Remind later]. Notifications are persisted in notifications.json (alongside sessions.json in bot-data/). Mode-2 /menu gains a "Notifications (N)" tab with unread badge, sitting alongside Sessions and Work. Actions: [Show] → full content as new message; [Delete] → remove from store, edit message; [Remind later] → one-shot re-fire in 1h (written as one-shot entry in schedules.json). Dialogue-interrupt concern is resolved by soft delivery — a compact notification is non-disruptive even mid-conversation; no quiet-window gating needed, only avoid firing while a response is actively streaming.
