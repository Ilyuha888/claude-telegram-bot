# Multi-user exclusion must be an explicit constraint, not implicit scope

## Rationale

The entire platform is scoped to a single user but this is nowhere stated as a hard constraint in the ADRs or architecture documents. Implicit scope is invisible to future agents and contributors who may inadvertently design for multi-user scenarios (e.g., shared vault, per-user session isolation, concurrent approval flows). The single-user constraint should be recorded explicitly so that any future decision to support multiple users triggers a deliberate architectural review, not an incremental drift.

## Evidence

README.md and ARCHITECTURE.md describe a personal assistant without stating single-user as a constraint. ADR-006 staging complexity is partly motivated by multi-editor risk avoidance, yet there is only one user.
