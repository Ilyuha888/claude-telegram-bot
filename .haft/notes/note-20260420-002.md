# Auto-resume after bot restart via tryAutoResume() in session.ts

## Rationale

After each bot process restart, this.sessionId was null in memory so query() fired without resume:, orphaning the prior session. The fix adds tryAutoResume() called at the start of sendMessageStreaming: reads getSessionList()[0], checks saved_at within 24h TTL and working_dir match, then sets this.sessionId. A standalone ctx.reply() notice is sent before the query. Verified 2026-04-20: journalctl shows Auto-resumed + RESUMING (not STARTING) on first post-restart message; notice renders in Telegram. Supersedes the misframed prob-20260420-004 (which assumed token-limit pause).

## Evidence
