# Log-outcome keyboard on scribe_reminder notifications

## Rationale

When a scribe_reminder fires, the notification keyboard shows [Log outcome] instead of [Show]. handleNewSession primes the new session with: "My reminder just fired: title\n\n{notif.content}\n\nI've done it. Read the note for context, then help me capture what I learned." notif.content includes the note_path, so the agent reads the original vault note and structures outcome capture. Alternative: generic "New session" button — rejected, scribe reminders need a distinct capture-outcome flow, not a generic chat opener.

## Affected Files

- `src/scheduler.ts`
- `src/handlers/mode2/notifications.ts`
