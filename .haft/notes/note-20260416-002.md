# Remote Control degradation accepted — SSH is the fallback when V_linuz90 is the Telegram bridge

## Rationale

With V_linuz90, Telegram messages are handled by the bot daemon's ephemeral Claude subprocesses, not the main Claude Code tmux session. Remote Control via claude.ai/code still works if the tmux session is kept running alongside the bot, but the two are independent contexts. Accepted degradation: Remote Control becomes a secondary concern. The primary fallback for admin/emergency access is SSH to the VM + tmux attach. This is sufficient because: (1) Remote Control was always a secondary channel, (2) SSH access is always available, (3) the scenario where Remote Control is needed but SSH isn't available is implausible for a single-user personal setup on a Hetzner VM.
