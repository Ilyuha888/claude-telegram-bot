# PKM skill intent routing added to SAFETY_PROMPT

## Rationale

Live test confirmed: /scribe fires correctly when invoked explicitly (Russian Cyrillic filename, correct frontmatter, mgrep duplicate check, commit-confirm). But natural-language capture intents ("Note for tomorrow: ...") bypassed skills and triggered ad-hoc vault writes. Fix: rule 6 in SAFETY_PROMPT instructs the model to invoke /scribe, /retriever, /curator from intent patterns rather than requiring explicit slash invocations. No code change — prompt-only. Closes prob-20260417-006.
