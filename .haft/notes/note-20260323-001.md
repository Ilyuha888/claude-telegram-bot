# Nested-tree helper required for Dulwich bare/memory repos

## Rationale

Dulwich porcelain (add, open_index) does not work on bare or MemoryRepo instances. The vault write path (dec-20260320-005) needs an app-level helper that converts flat file paths to nested Git trees bottom-up. A reference implementation exists in scripts/spike_dulwich_memory_commit.py (build_nested_tree function). Extract and adapt it when implementing the vault commit service. Key details: process directories deepest-first, set file mode 0o100644 explicitly, use 0o040000 for subtree entries.

## Evidence

Verified in spike: 16/16 checks passed with nested paths up to 4 levels deep (vault/notes/projects/alpha.md).

## Affected Files

- `scripts/spike_dulwich_memory_commit.py`
