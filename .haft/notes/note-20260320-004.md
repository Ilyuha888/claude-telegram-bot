# Testing strategy for LLM-dependent code paths is missing

## Rationale

No ADR or architecture document addresses how to test code paths that invoke the LLM. The options (live API calls in tests, recorded cassettes, mock adapters, deterministic stubs) have meaningfully different trade-offs in cost, flakiness, and coverage fidelity. Without a declared strategy, test files will be written inconsistently — some will hit the live API, some will not test the LLM layer at all. This should be decided before any tool-call or handler code is written.

## Evidence

No testing section exists in any ADR. ARCHITECTURE.md describes the LLMClient interface but does not mention testability or how the adapter boundary is exercised in CI.

## Affected Files

- `ARCHITECTURE.md`

## Deprecated (2026-04-15)

**Reason:** Testing strategy for LLM code paths was muscat-specific. No custom LLM code in the Claude Code on VM approach.
