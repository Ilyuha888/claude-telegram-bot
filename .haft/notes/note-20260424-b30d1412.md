# prob-20260416-006 signal revised — RC substrate exists; gap is time-trigger + dispatch layer only

## Rationale

prob-20260416-006 was framed ("ephemeral Claude subprocesses per conversation") before dec-20260422-003 shipped. That decision delivered a persistent tmux + claude remote-control substrate (verified 2026-04-23). The original signal is now obsolete on the "no persistent session" axis. Revised gap: there is no time-trigger mechanism (cron, calendar, in-process scheduler) and no dispatch path that fires a prompt at an RC session (or a short-lived claude -p run) on schedule, delivers the result to Telegram, and survives VM reboot. The problem is no longer about session persistence — it is about scheduled invocation on top of an already-working persistent substrate.
