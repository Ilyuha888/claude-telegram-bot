# Vault CLAUDE.md needs V-A methodology contract

## Signal

The vault CLAUDE.md currently reflects the thin navigation-and-zones contract from dec-20260416-002. It has no V-A-specific rules: no note type definitions, no lifecycle state conventions, no MOC navigation protocol, no frontmatter schema reference, no capture pipeline instructions. The agent will not reliably draft evergreen notes, create MOCs, or route captures correctly without these rules.

## Problem Type

synthesis

## Constraints

- CLAUDE.md must stay under 200 lines (dec-20260416-002 invariant)
- Must not break existing commit-confirm flow or zone write rules
- Must reference vault-index.md for navigation, not duplicate its content

## Optimization Targets

- Agent correctly drafts inbox captures into evergreen-format notes without explicit instruction
- Agent navigates topic queries via MOC → linked notes → grep fallback
- Agent applies frontmatter schema (type/status/created/source) to every new note

## Acceptance

Run the 3 canonical queries (vaccination, present idea, inference article) via Telegram after CLAUDE.md update. All 3 answered correctly with vault citations. Agent produces a correctly-formatted draft note from a raw voice memo without correction.

## Blast Radius

Vault repo only. CLAUDE.md is versioned — git revert restores previous behavior.
