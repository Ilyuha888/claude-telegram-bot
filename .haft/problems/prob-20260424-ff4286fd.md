# Google Calendar integration: schedule-aware agent context

## Signal

The agent has no awareness of the user's calendar. Daily focus routine fires at 09:00 MSK but has no context about what meetings/events are scheduled for the day. Scribe has no way to cross-reference time references against actual calendar slots. The user cannot ask "what's on my calendar today?" from Telegram.

## Problem Type

search

## Constraints

- Read-only by default — no event creation/modification without explicit confirmation
- OAuth credentials handled securely
- Calendar data must not appear in audit logs

## Optimization Targets

- Daily brief quality: agenda context in morning routine
- Scheduling awareness: Scribe can reference calendar when capturing time-bound notes

## Acceptance

Daily focus routine includes today's agenda (from Google Calendar) in its context. User can ask "what's on my calendar?" from Telegram and get a summary. Optionally: Scribe can detect scheduling intent and suggest calendar slots.

## Blast Radius

Daily focus scheduler prompt, Scribe skill (scheduling intent), potentially new /calendar command. Requires OAuth/MCP integration.

## Reversibility

high
