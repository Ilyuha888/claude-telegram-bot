# Choose a PKM methodology to give the vault operational semantics

## Signal

Infrastructure is live (dual vault, thin CLAUDE.md, vault-index.md, commit-confirm flow) but the vault has no methodology layer. Captures accumulate without a defined lifecycle. Notes are filed by role/topic but not linked, not typed in frontmatter, not distinguished by maturity. The agent can read and write files but has no contract for what kind of note to write, when to refactor, or how to surface staleness. Without a methodology the vault degrades into a filing cabinet — the collector's fallacy and orphan-note failure modes are already structurally guaranteed.

## Problem Type

synthesis

## Constraints

- Single-user scope (dec-20260320-001) — no multi-user patterns.
- Dual-vault architecture preserved (dec-20260416-002): User_Obsidian_Vault (human-authored) + Agent_Obsidian_Vault (agent-owned). Do not merge or restructure top-level.
- Commit-confirm gate on all user-vault writes is non-negotiable.
- Plain markdown + YAML frontmatter only — no proprietary formats, no separate database unless ROI is explicitly argued.
- Obsidian is the editing UI — methodology must work when notes are edited outside the agent.
- Must coexist with existing Cyrillic/English role structure in User_Obsidian_Vault — full restructure is out of scope.

## Optimization Targets

- Thinking-partner yield: methodology produces insight and original writing over years, not just organized capture.
- Agent retrieval quality: agent can reliably answer 'what did I think about X' without hallucination or excessive context burn.
- Curation cost: weekly maintenance under ~45 min; drift detectable by agent without human prompting.

## Observation Indicators

- Note count growth rate
- Ratio of human-authored to agent-drafted notes in User vault
- MOC count (too many = overhead; too few = navigational gaps)

## Acceptance

A chosen methodology documented as a haft decision, with: note types and lifecycle states defined, frontmatter schema (initial fields only), linking conventions (wikilinks vs tags vs MOCs vs Dataview), capture→refine→canonical pipeline traced end-to-end. Then a 2-week shakedown: 5 captures processed, 3 drafts promoted to evergreen, 1 MOC created or updated, zero human corrections of agent-misfiled notes.

## Blast Radius

Vault repo only — methodology changes affect CLAUDE.md in vault repo + new note templates. No runtime code. All changes version-controlled and rollbackable via git revert.
