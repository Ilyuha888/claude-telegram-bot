# Agent vault: is the User/Agent split load-bearing?

## Signal

User_Obsidian_Vault and Agent_Obsidian_Vault coexist in the knowledge base repo. The purpose of Agent_Obsidian_Vault is unclear: no vault skills write to it explicitly, Scribe targets User_Obsidian_Vault/00-inbox/, and Retriever searches both. It is unknown whether Agent_Obsidian_Vault contains any content that couldn't live in User_Obsidian_Vault.

## Problem Type

diagnosis

## Constraints

- Must not lose any user-authored notes
- Obsidian must remain able to open the vault without errors
- PKM skill paths must remain consistent after any restructure

## Optimization Targets

- Clarity: one obvious place to write new agent-generated content
- Skill simplicity: fewer path branches in Scribe/Retriever/Curator

## Observation Indicators

- Vault size and graph complexity
- Number of broken wikilinks after any restructure

## Acceptance

Decision: keep separate vaults with documented purpose boundary, OR merge into single vault with role-tagged content, OR delete Agent_Obsidian_Vault if empty/redundant. Either way: a written rationale and updated skill routing.

## Blast Radius

All three PKM skills (Scribe, Retriever, Curator). Any agent that writes vault content. Obsidian graph UI for the user.

## Reversibility

medium
