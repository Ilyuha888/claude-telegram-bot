# Bot message formatting and Telegram UX quality of life

## Signal

The Telegram bot currently sends plain text replies with no formatting conventions. Real-world usage reveals UX gaps: no distinction between informational responses and action confirmations, no structured formatting for vault search results, permission dialogs render as raw text, no message length management for long responses, and no conventions for code blocks, links, or structured data. Research evidence: Bot API 9.3 sendMessageDraft enables native streaming; MarkdownV2 formatting is controllable only via custom bot code, not the official plugin; community bots (linuz90, terranc, RichardAtCT) demonstrate mature formatting including numbered options → inline keyboard buttons.

## Constraints

- Telegram Bot API markdown subset (MarkdownV2 or HTML)
- Must work on mobile (small screen)
- 4096-character message limit

## Optimization Targets

- Readable, well-formatted bot responses in Telegram
- Clear visual distinction between query results, confirmations, and status messages

## Observation Indicators

- Message length distribution
- User re-asks due to unclear formatting

## Acceptance

Bot uses consistent formatting for: search results (bulleted with note paths), confirmations (clear yes/no prompt), status messages (emoji prefix), long responses (chunked or summarized). User can scan responses without reading walls of text.
