# Gmail integration: read and triage email from Telegram

## Signal

Emails that require action are not surfaced in the PKM/assistant workflow. The user has to context-switch to Gmail, manually triage, and then optionally capture to vault. Important items are missed or pile up. No mechanism exists to read email from Telegram, capture actionable emails as vault notes, or set reminders from email context.

## Problem Type

search

## Constraints

- Read-only by default — no send/reply without explicit confirmation
- OAuth credentials must not be stored in bot-data unencrypted
- Must not leak email content to logs

## Optimization Targets

- Friction reduction: zero context switch for email triage
- Vault capture: actionable emails become notes without copy-paste

## Acceptance

User can ask "what's in my inbox?" from Telegram and receive a triage summary. User can say "capture this email as a note" and Scribe creates a vault note. Optionally: agent proactively surfaces high-priority emails in daily focus routine.

## Blast Radius

Daily focus routine (scheduler-prompts.ts), Scribe skill, potentially new /gmail command or skill. Requires OAuth/MCP integration.

## Reversibility

high
