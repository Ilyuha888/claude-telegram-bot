# Telegram attachments (images, PDFs, documents) not accessible to agent

## Signal

When users send files via Telegram, the agent fails in two distinct ways. PDFs: Read tool returns "File does not exist" — confirmed with "Analyst_Ilya_Nartov (1).pdf" (2026-04-18). Images: agent hallucinates content, claims image is unclear despite high-res source, or fails to extract visible text. Root cause hypothesis: linuz90/claude-telegram-bot does not download attachments from Telegram Bot API and persist them to file paths the Claude Code agent can read. Agent receives message context but not a valid filesystem path to the file. Affects all attachment types (images, PDFs, documents). File pipeline investigation task: Agent_Obsidian_Vault/tasks/bot-improvement/2026-04-17-improve-image-handling.md. Dependency: dec-20260416-003 (linuz90 bot bridge is the affected codebase).

## Problem Type

diagnosis

## Constraints

- Fix must be in linuz90/claude-telegram-bot TypeScript/Bun codebase — no custom Python runtime (architecture invariant dec-20260414-001)
- File storage must be accessible to Claude Code agent on VM (under /home/assistant/ or /tmp/)
- Must not require bot restart for each attachment type
- Temp files must be cleaned up after session ends to avoid disk accumulation

## Optimization Targets

- attachment_access_reliability — agent can read 100% of files sent via Telegram
- vision_accuracy — agent correctly reads visible text and elements without hallucination

## Acceptance

Phase 1 (blocker): Agent can read PDFs sent via Telegram using Read tool; agent can access images sent via Telegram with valid filesystem paths; files persist long enough for agent to process them. Phase 2 (quality): Agent accurately reads text from high-res screenshots 90%+ of the time; agent never fabricates image content; when genuinely uncertain, asks user to clarify rather than hallucinating.

## Blast Radius

linuz90/claude-telegram-bot message handler and file download pipeline; no vault or VM security surface changes required. Fix is isolated to bot TypeScript code.

## Reversibility

high — adding file download and persistence to the bot is additive and easily reverted
