# Runtime and governance live in separate repos, creating context split for admin session

## Signal

The admin Claude session on the VM starts in the vault repo; haft decisions and bot source code live in two different Git repos, forcing context-switching and making it impossible to reason about code and architectural decisions in one working directory.

## Problem Type

optimization

## Optimization Targets

- single cwd that exposes both bot source and haft governance to the admin Claude session

## Acceptance

Admin Claude session starts from claude-telegram-bot repo root; /h-status returns active decisions; bot source and governance artifacts accessible without repo switching

## Blast Radius

Admin Claude session on VM; any agent working on bot features without governance context

## Reversibility

medium
