# Telegram bridge architecture: official plugin vs community Agent SDK bot

## Signal

The official Telegram channel plugin is live and working, but research confirms fundamental limitations that block three of four active UX problems: no message queuing (messages lost when session is closed), no voice transcription, dropped messages under stress, no retry logic, fire-and-forget architecture, no formatting control, single session only. Mature community alternatives (linuz90/claude-telegram-bot 423★, terranc/claude-telegram-bot-bridge 38★, hanxiao/claudecode-telegram 15★) demonstrate the CLI-auth/subscription-compatible bridge pattern with voice, multi-project, streaming, and inline keyboard approval flows. The admissibility constraint in dec-20260414-001 says to use the plugin first and fall back to Agent SDK only if the plugin proves unworkable. Evidence is accumulating that the plugin has a hard ceiling for the target UX quality.

## Problem Type

search

## Constraints

- Must use Claude subscription billing (Pro/Max) — not API pay-per-token; ~$10/mo budget for non-Claude API costs (voice transcription etc.)
- dec-20260414-001 admissibility: do not build custom bot unless plugin proves unworkable
- Single-user scope (dec-20260320-001)
- Telegram two-way messaging must remain operational during any transition
- Security posture must improve, not degrade — current setup runs as root

## Optimization Targets

- Message reliability (no dropped messages, queuing when session restarts)
- UX quality (formatting, streaming, voice transcription via Whisper ~$0.006/min)
- Permission model (canUseTool inline keyboard buttons vs static allowlist)

## Observation Indicators

- Implementation and maintenance effort
- Plugin evolution — does Anthropic improve the channel plugin to address these gaps?
- Agent SDK subscription billing policy stability (CLAUDE_CODE_OAUTH_TOKEN workaround)

## Acceptance

A decision record with evidence from either: (a) measured plugin failure rate over 2 weeks of real use, OR (b) a working prototype of the chosen alternative demonstrating the claimed improvements.

## Blast Radius

start-assistant.sh, systemd service, Telegram bot configuration, permission model. If switching: new bot code directory or adopted community repo.
