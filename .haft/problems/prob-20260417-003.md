# No agent skills for common PKM scenarios

## Signal

Three high-frequency PKM scenarios have no dedicated skill: (1) Scribe — capture Telegram voice/text into correctly-formatted inbox note; (2) Retriever — answer "what do I know about X" grounded in vault content; (3) Curator — generate weekly curation report (stale inbox, orphans, draft promotions, tag proposals). Without skills, the agent relies on CLAUDE.md instructions alone, which is fragile at the level of individual step execution.

## Problem Type

synthesis

## Constraints

- Skills must work from Telegram interface (short prompts, no file browser)
- Skills must respect commit-confirm gate for User_Obsidian_Vault writes
- Skills must not require external dependencies beyond existing Claude Code toolset

## Optimization Targets

- Scribe: voice memo → correctly-tagged inbox note in under 30 seconds
- Retriever: vault-grounded answer with note citations, no hallucination
- Curator: weekly report generated and delivered to Telegram proactively

## Acceptance

Each of the 3 skills invoked via Telegram produces correct output on first try for a real input, without user correction.

## Blast Radius

Skills are files in ~/.claude/skills/ — adding them has no effect on existing behavior. Removal reverts to CLAUDE.md-only operation.

## Reversibility

high
