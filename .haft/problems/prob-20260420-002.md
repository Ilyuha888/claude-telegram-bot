# No mechanism to spawn per-repo remote Claude Code sessions from Telegram

## Signal

The linuz90/claude-telegram-bot (dec-20260416-003) added /new and /resume for multi-session conversation within the knowledge-base repo, but cannot spawn Claude Code sessions bound to arbitrary repos or git worktrees on the VM. This is the unresolved multi-repo axis of prob-20260416-001, which was superseded by dec-20260416-003 only on the single-repo multi-session axis. The acceptance criterion from prob-20260416-001 — 'spawn a new session in a chosen repo, switch which session receives subsequent messages' — remains unmet. Mobile developers cannot initiate a focused coding session in e.g. data-style/junk_mart from Telegram and hand off to Cloud Code Remote on desktop. Detailed spec in Agent_Obsidian_Vault/tasks/2026-04-20-telegram-multi-repo-remote-session.md.

## Problem Type

synthesis

## Constraints

- VM SSH access required at dispatch time (repos under ~/repos/)
- Corporate subscription tier must be active before Opus-plan-mode sessions can be used
- Worktree must never be deleted by /close — only detached
- Failed dispatch must clean up any partially-created worktrees
- No LLM reasoning in Mode-2 command layer (menu-only)

## Optimization Targets

- session_handoff_latency — time from /work command to usable Cloud Code Remote session
- worktree_integrity — zero orphaned worktrees under normal and error paths

## Observation Indicators

- Worktree count growth over time (watch for orphan accumulation)
- VM disk usage under ~/repos/ (session state must not consume unbounded storage)

## Acceptance

/work <repo> [path] [worktree] [branch] spawns a remote Claude Code session on VM and returns a session id; Cloud Code Remote can attach; /sessions lists active sessions; /attach <id> routes subsequent messages; /close <id> detaches without deleting git worktree; 7-day idle triggers auto-close while preserving worktree state; zero dangling git worktrees after any edge-case disconnection.

## Blast Radius

linuz90 bot command layer, VM ~/repos/ tree, systemd session management, CLAUDE.md per-repo routing; does not affect Obsidian Sync or vault contents. Reopens the multi-repo axis of prob-20260416-001 (superseded on multi-session axis by dec-20260416-003).

## Reversibility

medium — each session independently reversible; worktrees easily pruned; bot command surface change is rollback-safe
