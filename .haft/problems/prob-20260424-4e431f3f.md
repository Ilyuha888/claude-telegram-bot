# Vault structure and note lifecycle undefined — skills target inconsistent paths

## Signal

Agent_Obsidian_Vault has 5 non-empty folders (tasks/28 files, indexes/2, reviews/2, rules/1, profile/1) and 3 empty ones (notes, drafts, references). weekly_curator reads Agent_Obsidian_Vault/drafts/ in every run but it is always empty. User_Obsidian_Vault has 30-areas/ (9 files) and 50-templates/ (0 files) referenced by no skill or routine. No process doc exists for the note lifecycle: how a capture in 00-inbox/ graduates to 10-notes/, who promotes it and when, and what criteria trigger archiving. Skill prompts target inconsistent vault subsets with no canonical spec.

## Problem Type

synthesis

## Constraints

- Must not lose any existing notes
- Obsidian graph must remain valid after any restructure
- Skill changes must not break existing prompt_key scheduler routines

## Optimization Targets

- Clear inbox→stable-note lifecycle with unambiguous trigger conditions

## Observation Indicators

- Inbox backlog size over time
- Number of vault folders referenced by zero skills (dead paths)

## Acceptance

Written spec of vault structure (folder purposes, content types, note lifecycle stages). All skill prompts and scheduler routines reference folders consistently with the spec. Dead paths eliminated. User knows what goes where.
