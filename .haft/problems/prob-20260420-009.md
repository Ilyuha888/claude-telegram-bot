# Telegram message context enrichment — pass forward/reply/quote source to Claude

## Signal

Source audit of the live linuz90 bot (note-20260420-003..006, evid-20260420-428532000) confirmed all 5 delivery scenarios are *safe* but three of them are *UX-incomplete*: forwarded messages lose their source attribution, reply-to drops the original message that gives the reply meaning, and Bot-API 7.0 `message.quote` fragments never reach Claude. The handler reads only `ctx.message.text` / `ctx.message.caption` — all provenance metadata (`forward_origin`, `reply_to_message`, `quote`) is silently discarded. User policy calls: (1) forward → include source + content; (2) edit → keep silent-drop; (3) rapid succession → keep sequentialize; (4) reply-to → include source (original message); (5) quote → include quoted fragment.

## Problem Type

synthesis

## Constraints

- No code changes to edit-handling (scenario 2) or sequentialize middleware (scenario 3)
- Reply-to payload must be truncated to bound context size (budget: ~500 chars of original text)
- No new handlers — enrich inside existing text.ts and media handlers only
- Must not break existing plain-text message path — enrichment only fires when metadata present

## Optimization Targets

- context completeness: Claude sees source/provenance for forwards, replies, and quotes
- implementation compactness: one helper function, minimal blast radius

## Observation Indicators

- total prompt size sent to Claude (watch for reply-to bloat)
- Claude response quality on reply-to flows (subjective)

## Acceptance

Handler enriches user text with forward_origin source line (when present), reply_to_message truncated snippet (when present), and message.quote fragment (when present), before passing to session.sendMessageStreaming. Verified by: sending a forward, a reply, and a quoted message from Telegram, reading journald to confirm the enriched string reaches the query() call. Edit and rapid-succession behavior unchanged.

## Blast Radius

linuz90 bot src/handlers/text.ts (primary); optionally src/handlers/photo.ts, document.ts, audio.ts, video.ts, voice.ts for forward/reply on media captions. No Claude Code SDK, session layer, or MCP changes.
