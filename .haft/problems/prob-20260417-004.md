# Knowledge Base initial migration to V-A structure

## Signal

The vault currently has notes filed by role/topic (Cyrillic structure) with no frontmatter schema, no lifecycle states, no templates, no MOCs, and no standardized naming conventions. Before the V-A methodology (dec-20260417-001) can operate, the structural prerequisites need to exist: folder layout, templates, initial topic index, and at minimum one seed MOC and a few evergreen notes to establish the pattern.

## Problem Type

synthesis

## Constraints

- Existing notes must not be deleted — migrate or leave in place, never destroy
- Migration must be done on VM via SSH or Telegram, not by directly editing vault locally
- Blog import goes to Agent_Obsidian_Vault/references/ — not User_Obsidian_Vault

## Optimization Targets

- Folder layout matches dec-20260417-001 spec (00-inbox through 60-mocs + templates)
- Note templates exist for: evergreen, inbox, project, person
- At least 5 seed notes written in correct evergreen format with frontmatter + 3 wikilinks each
- Agent_Obsidian_Vault/indexes/topic-index.md seeded from existing vault content

## Acceptance

Agent can navigate from topic-index.md to a MOC to a linked evergreen note in a single query. One end-to-end capture \u2192 draft \u2192 promote cycle completed successfully.

## Blast Radius
