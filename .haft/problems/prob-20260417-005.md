# No routine model for KB usage — when and how to interact

## Signal

V-A requires a weekly curation review, proactive capture habits, and periodic promotion of drafts to evergreen. None of this happens automatically or is scheduled. Without an explicit routine — what triggers the weekly review, what the agent does proactively vs reactively, when blog writing sessions happen — the methodology will decay into ad-hoc capture with no processing. Related to prob-20260416-006 (no scheduled task execution from Telegram).

## Problem Type

synthesis

## Constraints

- Routine must be triggerable via existing infrastructure (Telegram, cron on VM, or Google Calendar MCP)
- Routine must not require daily manual effort — agent handles continuous background work
- Proactive reminders must be non-intrusive and batchable

## Optimization Targets

- Weekly curation report delivered to Telegram proactively (Sunday evening)
- Capture habit reinforced without friction — any Telegram message can seed a note
- Monthly and quarterly reviews scheduled and prompted by agent

## Observation Indicators

- How often user initiates capture unprompted vs only when agent reminds
- Whether weekly review is completed within 24h of delivery

## Acceptance

Weekly curation report delivered automatically for 4 consecutive weeks. User completes review within 24h at least 3 of 4 times. Monthly review scheduled and completed once.

## Blast Radius

VM cron + Telegram bot. No vault content affected. Reminders can be disabled by removing cron job.

## Reversibility

high
