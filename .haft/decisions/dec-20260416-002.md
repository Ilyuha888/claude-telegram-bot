# Thin CLAUDE.md + Generated Index + Agent Zone Bootstrap

## 1. Problem Frame

**Signal:** The vault CLAUDE.md was written before the assistant went live. First real-world tests revealed gaps: Claude didn't auto-commit after writes (now fixed with confirm flow), permission prompts leaked into Telegram (fixed with defaultMode override), and the navigation guide was written from structure analysis without testing actual assistant queries. The CLAUDE.md likely needs tuning for: search strategies that actually work via Telegram, better instructions for handling Cyrillic path search, clearer agent zone conventions, and response formatting guidance.

**Constraints:**
- Do not break existing commit-confirm flow
- Keep CLAUDE.md under 200 lines

**Acceptance:** Run 10 representative vault queries via Telegram (search, cross-reference, create note, edit note). Claude handles all correctly without user intervention beyond the confirm step.

## 2. Decision

**Selected:** Thin CLAUDE.md + Generated Index + Agent Zone Bootstrap

**Selection policy:** Maximize agent zone readiness and drift resistance; provide deterministic Cyrillic/English bridging; keep CLAUDE.md under 200 lines

**Why selected:** V-B is the sole Pareto-dominant variant. It scores High across all five comparison dimensions. The vault-index.md externalizes navigational knowledge into a regenerable artifact, solving both drift resistance and the 200-line budget. Bootstrapping the agent zone with a real index artifact fulfills the user requirement for full agent zone coverage and establishes the pattern for subsequent agent artifacts (tasks, reviews, drafts).


**Invariants:**
- Commit-confirm flow must not be bypassed for any vault write
- CLAUDE.md must stay under 200 lines
- Single-user scope (dec-20260320-001) — no multi-user patterns in instructions
- Agent artifacts go only into Agent_Obsidian_Vault/ — never into User_Obsidian_Vault/

**Pre-conditions:**
- [ ] vault-index.md is generated and committed before CLAUDE.md goes live
- [ ] SSH access to VM is verified
- [ ] Existing commit-confirm flow tested and working

**Post-conditions:**
- [ ] CLAUDE.md revised to ~130 lines, referencing vault-index.md for navigation
- [ ] Agent_Obsidian_Vault/indexes/vault-index.md exists with complete folder map and English aliases
- [ ] 12-query acceptance test passes: 6 user-facing, 4 agent-facing, 2 edge cases
- [ ] No user intervention required beyond commit-confirm step

## 3. Rationale

**Counterargument:** V-A (inline routing table) avoids the two-file dependency and is entirely self-contained. If the vault-index.md is ever deleted or forgotten, CLAUDE.md becomes under-specified. The regeneration step adds friction that may not happen reliably.

**Selected variant weakest link:** vault-index.md availability — missing or stale index degrades navigation to unguided mgrep with no Cyrillic bridging

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| Thin CLAUDE.md + Generated Index + Agent Zone Bootstrap | **Selected** | V-B is the sole Pareto-dominant variant. It scores High a... |
| V-A — Inline Routing Table | Rejected | Dominated by V-B: same query bridging quality but Low drift resistance (static table) and tight ~185-line budget. No stepping-stone value. |
| V-C — Semantic Search-First | Rejected | Unknown Cyrillic/English bridging quality — depends entirely on untested mgrep semantic matching for Cyrillic paths. Dominated by V-B. |

**Evidence requirements:**
- Read CLAUDE.md after revision — confirm line count under 200 and no Glob/Grep references
- Read vault-index.md — confirm all 5 role domains and key subfolders mapped
- Run 12-query Telegram acceptance test — confirm 0 unexpected corrections

**Predictions:**
| Claim | Observable | Threshold |
|-------|------------|-----------|
| CLAUDE.md instructs mgrep + vault-index.md lookup, replacing vague Glob/Grep references | Read CLAUDE.md on VM — confirm no Glob/Grep references, confirm mgrep + index instructions present | Zero Glob/Grep references; mgrep mentioned in search strategy |
| vault-index.md provides English aliases covering all 5 user role domains and key hard-skill subfolders | Read vault-index.md — count folders covered and verify English aliases for БД, Математика, LLM, Python, Я аналитик, Я студент | All 5 role domains + 6 hard-skill subfolders mapped with ≥2 English aliases each |
| Agent zone workflow guide enables correct note-taking, draft, and index patterns without user correction | Run 4 agent-facing test queries via Telegram; count corrections required | 0 corrections required beyond commit-confirm step |

## 4. Consequences

**Rollback plan:**
Triggers:
- vault-index.md goes missing and regeneration fails
- CLAUDE.md revision causes more user corrections than before
Steps:
1. git revert the CLAUDE.md commit on the vault repo
2. Delete vault-index.md if it was committed
3. Restore previous CLAUDE.md from git history
Blast radius: vault repo only — no runtime code affected

**Refresh triggers:**
- Vault structure changes significantly (new top-level folders, role domains added)
- User corrections to vault behavior exceed 3 in a week after this ships
- mgrep gains reliable Cyrillic semantic matching — V-C becomes viable, vault-index.md may be reducible

**Affected files:** ~/repos/my_obsidian_knowledge_base/CLAUDE.md, ~/repos/my_obsidian_knowledge_base/Agent_Obsidian_Vault/indexes/vault-index.md
