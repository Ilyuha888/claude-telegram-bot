# Merge governance repo into bot fork via git subtree

## 1. Problem Frame

**Signal:** The admin Claude session on the VM starts in the vault repo. Haft decisions and bot source code lived in two different Git repos (`yet_another_personal_assistant` + `claude-telegram-bot`), forcing context-switching and making it impossible to reason about code and architectural decisions in one working directory.

**Constraints:**
- Preserve full haft commit history — audit trail must remain reachable via git log
- Upstream fork relationship with linuz90/claude-telegram-bot must remain intact
- The Obsidian knowledge vault stays in its own separate Git repo
- `.mcp.json` must work on both Mac and Hetzner VM without per-host edits

**Acceptance:** `/h-status` returns 7+ active decisions from `claude-telegram-bot` cwd; bot source accessible alongside governance artifacts; upstream merge dry-run shows zero conflicts on governance paths.

## 2. Decision

**Selected:** Merge governance repo into bot fork via git subtree

**Selection policy:** Preserve history + minimal upstream conflict surface + single cwd for the admin session.

**Why selected:** `git subtree add --prefix=governance` imports the full commit history of `yet_another_personal_assistant` into the fork's DAG without squashing. Governance paths (`.haft/`, `AGENTS.md`, `.mcp.json`, `governance/`) are disjoint from upstream's file tree — zero merge conflict surface. The old repo is archived after the merge. A single cwd eliminates the context split.

**Invariants:**
- The Obsidian knowledge vault remains in its own separate Git repo — this decision does not change that invariant
- Bot runtime code in `src/` continues to track upstream `linuz90/claude-telegram-bot`
- `.mcp.json` uses `command: "haft"` (PATH-relative) and no `HAFT_PROJECT_ROOT` (cwd default)

**Pre-conditions:**
- [x] `yet_another_personal_assistant` HEAD is clean (no uncommitted haft changes)
- [x] Upstream `linuz90/claude-telegram-bot` has no `.haft/`, `AGENTS.md`, or `.mcp.json`

**Post-conditions:**
- [x] `.haft/` at repo root, discovered by haft MCP via cwd default
- [x] `.mcp.json` at repo root, portable across Mac and VM
- [x] `AGENTS.md` (governance content) at root; `CLAUDE.md` (bot docs) preserved with governance pointer appended
- [x] `governance/` subtree contains ARCHITECTURE.md, docs/, scripts/
- [x] `yet_another_personal_assistant` archived on GitHub
- [ ] VM admin session (`start-assistant.sh`) starts from `claude-telegram-bot` repo root

**Admissibility:**
- NOT: Must not squash the subtree history — haft audit trail must remain reachable via `git log`
- NOT: Must not store `HAFT_PROJECT_ROOT` as an absolute Mac path in `.mcp.json`

## 3. Rationale

**Counterargument:** Upstream merges from linuz90 must not clobber governance-only paths. Conflict surface is near-zero today but requires ongoing vigilance — if upstream ever adds `AGENTS.md`, `CLAUDE.md`, or `.mcp.json`, a merge would produce a conflict that could silently lose the governance pointer.

**Selected variant weakest link:** If upstream adds a `CLAUDE.md` or `AGENTS.md` in a future release, a merge produces a conflict that could lose the governance pointer appended to `CLAUDE.md`.

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| Keep repos separate | Rejected | Admin session can't reason about code and decisions together; haft artifacts invisible when working in bot repo |
| Bot as submodule of governance repo | Rejected | Complicates upstream sync — rebasing from linuz90 entangles with submodule pointer updates |

**Predictions:**
| Claim | Observable | Threshold |
|-------|------------|-----------|
| haft MCP works from bot repo root | `/h-status` returns active decision list from `claude-telegram-bot` cwd | All 7 expected active decisions visible within 30s of session start |

## 4. Consequences

**Rollback plan:**
Triggers:
- Upstream merge conflict on governance paths
- haft MCP fails to start from bot repo root
Steps:
1. `git revert` the subtree add commit (reverts governance/ import)
2. Restore `yet_another_personal_assistant` from archive on GitHub
3. Revert `.mcp.json` and `AGENTS.md` to pre-merge state
Blast radius: Governance paths only (`.haft/`, `AGENTS.md`, `.mcp.json`, `governance/`) — bot source in `src/` is untouched

**Refresh triggers:**
- upstream linuz90 adds `AGENTS.md`, `CLAUDE.md`, or `.mcp.json`
- A second governance concern emerges that warrants its own repo
- haft is removed from the project

**Affected files:** AGENTS.md, CLAUDE.md, .mcp.json, .haft/project.yaml, governance/ARCHITECTURE.md, governance/docs/, governance/scripts/
