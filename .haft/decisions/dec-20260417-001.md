# Evergreen-MOC-GTD Hybrid

## 1. Problem Frame

**Signal:** Infrastructure is live (dual vault, thin CLAUDE.md, vault-index.md, commit-confirm flow) but the vault has no methodology layer. Captures accumulate without a defined lifecycle. Notes are filed by role/topic but not linked, not typed in frontmatter, not distinguished by maturity. The agent can read and write files but has no contract for what kind of note to write, when to refactor, or how to surface staleness. Without a methodology the vault degrades into a filing cabinet — the collector's fallacy and orphan-note failure modes are already structurally guaranteed.

**Constraints:**
- Single-user scope (dec-20260320-001) — no multi-user patterns.
- Dual-vault architecture preserved (dec-20260416-002): User_Obsidian_Vault (human-authored) + Agent_Obsidian_Vault (agent-owned). Do not merge or restructure top-level.
- Commit-confirm gate on all user-vault writes is non-negotiable.
- Plain markdown + YAML frontmatter only — no proprietary formats, no separate database unless ROI is explicitly argued.
- Obsidian is the editing UI — methodology must work when notes are edited outside the agent.
- Must coexist with existing Cyrillic/English role structure in User_Obsidian_Vault — full restructure is out of scope.

**Acceptance:** A chosen methodology documented as a haft decision, with: note types and lifecycle states defined, frontmatter schema (initial fields only), linking conventions (wikilinks vs tags vs MOCs vs Dataview), capture→refine→canonical pipeline traced end-to-end. Then a 2-week shakedown: 5 captures processed, 3 drafts promoted to evergreen, 1 MOC created or updated, zero human corrections of agent-misfiled notes.

## 2. Decision

**Selected:** Evergreen-MOC-GTD Hybrid

**Selection policy:** Maximize thinking-partner yield first; maximize agent retrieval fit second; minimize curation burden third.

**Why selected:** Wins on primary target (thinking-partner yield: high) with the highest 5-year survival rate in practitioner literature. MOCs give the agent structured entry points — the single most important structure for reliable retrieval at vault scale. Compatible with an agent-heavy capture pipeline where the human captures raw and the agent drafts notes for weekly review. Blog import + ideas-as-seeds writing flow fits naturally: agent indexes the archive, surfaces relevant posts when developing new ideas.


**Invariants:**
- Agent never rewrites human-authored notes in User_Obsidian_Vault in place — drafts go to Agent_Obsidian_Vault/drafts/ for human review
- Commit-confirm gate on all User_Obsidian_Vault writes (except 00-inbox/ for raw capture)
- Dual-vault architecture preserved: User_Obsidian_Vault (human-authored) + Agent_Obsidian_Vault (agent-owned)
- Plain markdown + YAML frontmatter only — no proprietary formats, no separate database

**Pre-conditions:**
- [ ] Frontmatter schema defined: type, status, created, source (minimum fields)
- [ ] Note templates created for evergreen, inbox, project, person note types
- [ ] Vault CLAUDE.md updated with V-A navigation contract and write rules
- [ ] Blog archive import plan established (target folder in Agent_Obsidian_Vault/references/)

**Post-conditions:**
- [ ] 5 Telegram captures processed through full inbox→draft→evergreen pipeline
- [ ] 3 agent-drafted notes reviewed and promoted to User_Obsidian_Vault/10-notes/
- [ ] 1 MOC created or updated for a real topic cluster
- [ ] 0 human corrections of agent-misfiled notes in first 2 weeks
- [ ] Agent can correctly answer the 3 canonical queries: vaccination status, present idea for wife, article on model inference

## 3. Rationale

**Counterargument:** V-D (staged probe) remains the honest alternative. Without prior usage data, MOC creation discipline is a theoretical commitment. If the weekly review habit decays or MOC maintenance feels like overhead after 2 months, V-D would have been the better entry point.

**Selected variant weakest link:** MOC maintenance discipline — if MOCs fall stale, agent retrieval degrades silently and the navigational advantage over raw grep collapses

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| Evergreen-MOC-GTD Hybrid | **Selected** | Wins on primary target (thinking-partner yield: high) wit... |
| Karpathy LLM-Wiki Pattern (V-B) | Rejected | Sacrifices thinking-partner yield — human authors only raw fragments, agent compiles everything. Semantic drift over iterative rewrites. Defeats the blog writing goal where the user's own ideas need to be the seeds. |
| PARA + Zettelkasten Split (V-C) | Rejected | PARA's 3-year collapse from Resource/Area boundary ambiguity is structural, not a setup problem. Would require full restructure of existing vault for a methodology with documented abandonment rate. |
| Staged Minimal Substrate → V-A (V-D) | Rejected | The probe strategy was primarily justified by methodology-hopping risk. User has never run a PKM system before — no hopping history. Cold commitment to V-A is safe. V-D's substrate is identical to V-A's first step, so nothing is lost by committing now. |

**Predictions:**
| Claim | Observable | Threshold |
|-------|------------|-----------|
| Weekly 30-min curation review is sustainable with agent-generated report | User completes 4 consecutive weekly reviews without skipping | 4/4 reviews completed in first month |
| Agent retrieval answers the 3 scenario queries correctly without hallucination | Ask agent each of the 3 canonical queries after 2 weeks of captures | All 3 answered correctly with vault citations, zero hallucinated facts |
| Blog import creates usable reference corpus for writing assistance | Ask agent to suggest related past posts when developing a new idea | Agent surfaces at least 1 genuinely relevant past post per new idea session |

## 4. Consequences

**Rollback plan:**
Triggers:
- Weekly review habit decays for 4+ weeks and agent MOCs are stale
- Vault feels like a filing cabinet after 3 months of use
Steps:
1. Freeze new note creation for 1 week
2. Run orphan report — identify what has no links
3. Migrate to V-D substrate (keep frontmatter + lifecycle, drop MOC discipline)
4. Reopen prob-20260417-001 and re-decide with 3 months of observed usage data
Blast radius: Vault repo only — all notes are plain markdown, migration is mechanical

**Refresh triggers:**
- Vault exceeds 1000 notes with no MOC updates in 3 months — MOC discipline has broken down
- Capture volume exceeds 10/day consistently — inbox floods, collector's fallacy risk
- Weekly review habit decays for 4+ consecutive weeks — consider V-D retroactively
- Blog writing use case matures enough to warrant dedicated writing workflow decision
