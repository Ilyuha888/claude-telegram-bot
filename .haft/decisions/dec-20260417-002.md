# Thin rules in CLAUDE.md + va-contract.md

## 1. Problem Frame

**Signal:** The vault CLAUDE.md currently reflects the thin navigation-and-zones contract from dec-20260416-002. It has no V-A-specific rules: no note type definitions, no lifecycle state conventions, no MOC navigation protocol, no frontmatter schema reference, no capture pipeline instructions. The agent will not reliably draft evergreen notes, create MOCs, or route captures correctly without these rules.

**Constraints:**
- CLAUDE.md must stay under 200 lines (dec-20260416-002 invariant)
- Must not break existing commit-confirm flow or zone write rules
- Must reference vault-index.md for navigation, not duplicate its content

**Acceptance:** Run the 3 canonical queries (vaccination, present idea, inference article) via Telegram after CLAUDE.md update. All 3 answered correctly with vault citations. Agent produces a correctly-formatted draft note from a raw voice memo without correction.

## 2. Decision

**Selected:** Thin rules in CLAUDE.md + va-contract.md

**Selection policy:** Maximize agent correctness first; maximize line headroom second; confirmed by prompt engineering principle 11 (keep system prompts focused — externalize reference spec, keep CLAUDE.md as policy layer).

**Why selected:** Correctness matches V-A (both HIGH) via proactive-read mechanism already proven for tasks/ and profile/. Leaves 28 lines of headroom vs 10 for V-A — material given the vault CLAUDE.md has iterated twice from Telegram friction signals. Prompt engineering principle 11 confirms: note-type enums and lifecycle definitions are reference spec, not policy — they belong in va-contract.md, not CLAUDE.md. Mirrors the established vault-index.md pattern from dec-20260416-002.


**Invariants:**

- Commit-confirm gate unchanged for all User_Obsidian_Vault writes
- Agent artifacts go only into Agent_Obsidian_Vault/

## 3. Rationale

**Counterargument:** V-A (full inline) avoids the two-hop read entirely — rules and definitions always in scope at zero dependency cost. If the proactive-read mechanism degrades (mgrep quota exhausted, session context trimmed), V-B silently loses its definition layer while appearing to work. This failure mode is invisible without inspection.

**Selected variant weakest link:** va-contract.md proactive read — if the Proactive Zone Reads rule doesn't fire before a note operation, the agent has behavioral rules but no type/lifecycle definitions to apply them against

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| Thin rules in CLAUDE.md + va-contract.md | **Selected** | Correctness matches V-A (both HIGH) via proactive-read me... |
| Full inline V-A section | Rejected | Principle 11 violation: reference spec crowding the policy layer. 10-line headroom will close after the next 1-2 friction signals. Functionally equivalent on correctness. |
| Compressed inline only (~12 lines) | Rejected | Medium correctness fails the acceptance criterion — agent must apply V-A rules without explicit instruction; compressed rules misfire on ambiguous edge cases. |
