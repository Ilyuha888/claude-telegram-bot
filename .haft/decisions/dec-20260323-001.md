# C — Hybrid: skeleton + deferred

## 1. Problem Frame

**Signal:** ADR-008 specifies exact session state fields before any implementation exists. The field `active_facts` is typed as a free-form string list, violating the ADR's own rule about typed boundaries. The field `decisions` name-collides with the quint artifact namespace. Premature field locking without a running system invites schema churn.

**Constraints:**
- Session schema must use explicit typing and Pydantic (project convention)
- Field names must not collide with established quint or domain vocabulary
- Must not store secrets in session state

## 2. Decision

**Selected:** C — Hybrid: skeleton + deferred

Retains design communication for stable fields while deferring speculative ones. Eliminates the active_facts free-form string anti-pattern and the decisions namespace collision. Low rollback cost — only provisional fields need revision during prototyping.

**Invariants:**
- schema_version is mandatory and must be incremented on incompatible shape changes
- No bare string collections for structured session fields (pending_tasks, open_loops, workspace_rulings, or future additions)
- Session field names must not collide with quint artifact namespaces or established domain vocabulary
- New session fields must be typed Pydantic models before they enter the normative schema

**Pre-conditions:**
- [ ] No production code exists yet — this is a docs-only change
- [ ] ADR-008 is the current accepted schema specification

**Post-conditions:**
- [ ] ADR-008 status is Superseded with reference to this decision
- [ ] ARCHITECTURE.md session YAML has no active_facts field
- [ ] ARCHITECTURE.md session YAML uses workspace_rulings instead of decisions
- [ ] Normative rules updated to reflect deferred fields and addition principles

**Admissibility:**
- NOT: Free-form list[str] for any structured session collection
- NOT: Field names that collide with .quint/ directory names (decisions, problems, notes, solutions)
- NOT: Adding new fields to the normative schema without a typed Pydantic contract
